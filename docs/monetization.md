# Monetization Plan

Payments should come after the core game is fun and replayable.

## Free tier

Recommended first free tier:

- limited daily games
- Classic Mode
- basic characters
- basic result card

## Premium tier ideas

Possible premium benefits:

- higher daily game limit
- Detective Mode
- Hidden Role Mode
- Chaos Mode
- Daily Challenge archive
- premium themes
- no-watermark result cards
- advanced post-game analysis

## Pricing experiments

Possible experiments later:

- monthly membership
- small one-time game packs
- theme packs
- character packs

Do not implement these before retention is proven.

## What not to do early

Avoid:

- blocking the first game behind payment
- interrupting gameplay with monetization prompts
- building payment code before game quality is proven
- creating too many plans before knowing user demand

## Cost control

Use model tiers:

- cheaper model for free games
- stronger model for premium modes
- strict output token limits
- one Game Director call per phase in MVP

Track:

- cost per game
- tokens per completed game
- completion rate
- premium interest clicks
- share clicks

## Best conversion moments

Good places to show premium later:

- after a completed game
- when user tries a locked mode
- after daily free limit is reached
- on result page with no-watermark card option

## Brutal monetization truth

People will not pay because the app uses AI.

They might pay if:

- the game is genuinely fun
- modes feel different
- characters are memorable
- result cards are cool
- premium gives replay value
