# AI Agents Design

AI characters are the heart of Trust No Bot. The game will fail if every character sounds the same.

## Agent layers

Each AI player should have four layers.

### 1. Public identity

Visible to the user:

- name
- avatar/color
- short personality label
- alive/dead status

Example:

```json
{
  "name": "Arjun",
  "publicStyle": "confident, direct, impatient",
  "avatarKey": "arjun"
}
```

### 2. Secret role

Hidden until reveal:

- role
- team
- special action availability

This must be stored by the deterministic game engine.

### 3. Private memory

Compact notes used for future dialogue:

- who accused them
- who defended them
- who changed votes
- contradictions noticed
- private role goals

Never send unlimited full transcript forever. Summarize.

### 4. Public speech

Generated by the Game Director and shown in transcript.

Speech should match:

- personality
- current suspicion
- public facts
- private role goal
- prior memory

## Personality parameters

Recommended numeric fields from 0 to 1:

```ts
type AgentTraits = {
  aggression: number;
  caution: number;
  humor: number;
  logic: number;
  empathy: number;
  defensiveness: number;
  riskTolerance: number;
  talkativeness: number;
};
```

## Suspicion map

Each agent can maintain suspicion toward every other alive player.

```ts
type SuspicionMap = Record<PlayerId, number>; // 0 to 1
```

Suspicion should change based on:

- accusations
- defense patterns
- voting behavior
- contradictions
- night outcome
- direct answers
- silence

## Memory shape

Use compact memory.

```ts
type AgentMemory = {
  publicNotes: string[];
  privateNotes: string[];
  recentAccusations: Array<{ from: PlayerId; target: PlayerId; reason: string }>;
  voteHistory: Array<{ day: number; votedFor: PlayerId; reason?: string }>;
};
```

## Important guardrails

- Agents must not reveal hidden roles directly before game over.
- Agents can be suspicious, defensive, funny, or evasive.
- Agents should not use slurs, threats, or targeted real-world abuse.
- Agents should not mention prompts, models, or hidden system messages.
- Agents should not claim deterministic game events unless the engine provides them.

## MVP implementation shortcut

Do not run one model call per agent in MVP.

Use one Game Director call that writes messages for multiple agents while respecting their profiles. This is cheaper and easier to control.

Later, premium mode can experiment with one call per agent.
